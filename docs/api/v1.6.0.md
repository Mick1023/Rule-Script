# RuleScript v1.6.0 API Reference

## `RuleScriptFormatter`

Namespace: `RuleScript.Core.Formatting`

```csharp
public static class RuleScriptFormatter
{
    public static string Format(string source);
}
```

`Format` parses and formats valid Rule Script source. It returns LF-terminated text with four-space indentation, canonical token spacing, normalized block layout, preserved blank lines, and preserved comments/directives. Standalone comments follow the indentation of the next syntax element, while multi-line comment content is indented beneath its delimiters. Invalid source retains the normal lexer or parser `SyntaxException` behavior. Passing `null` throws `ArgumentNullException`.

Formatting is idempotent: formatting already formatted source returns the same text.

## `RuleScriptLanguageService`

Namespace: `RuleScript.Core`

### `GetRegions`

```csharp
public static IReadOnlyList<RuleScriptRegion> GetRegions(string source);
```

Parses `#region` and `#endregion` pairs and returns them in source order. Nested regions are supported. Source coordinates are one-based; `EndColumn` is exclusive.

### `GetFunctionDocumentation`

```csharp
public static string? GetFunctionDocumentation(
    string source,
    string functionName);
```

Returns normalized documentation for the named user function, or `null` if the function is missing or has no documentation. Empty delimiter lines at the beginning and end of a documentation block are removed; retained lines are joined with `\n`.

## `RuleScriptRegion`

Namespace: `RuleScript.Core.Parser`

```csharp
public sealed record RuleScriptRegion(
    string Name,
    int StartLine,
    int StartColumn,
    int EndLine,
    int EndColumn);
```

`Parser.Regions` exposes the same metadata after construction and parsing. Region directives are trivia and are removed from the executable parser token stream.

## Unified Function Symbols

Namespace: `RuleScript.Core.Runtime`

```csharp
public IReadOnlyList<RuleScriptFunctionSymbol> RuleScriptAnalysisResult.Functions { get; }
public RuleScriptFunctionKind RuleScriptFunctionSymbol.Kind { get; }
public string? Documentation { get; }
```

`RuleScriptAnalysisResult.Functions` is the preferred editor-facing function lookup surface. It returns one symbol model for user-defined, imported, host, and built-in functions. Use `RuleScriptFunctionSymbol.Kind` to distinguish `User`, `Imported`, `Host`, and `Builtin` origins.

`RuleScriptFunctionSymbol` carries parameter metadata, return type, documentation, source location/range when available, and origin-specific metadata:

- `HostMetadata` for host execution metadata such as async, thread-safety, and variadic behavior.
- `BuiltinMetadata` for built-in function entries.
- `ImportMetadata` for imported function source path, alias, and original name.

The `Documentation` property contains the `///` block associated with a user-defined or imported function, or registered documentation for host and built-in functions. Legacy `UserFunctions`, `HostFunctions`, and `BuiltinFunctions` collections remain available as compatibility projections, but new editor features should prefer `Functions`.

## Host Function Documentation and Options

Namespace: `RuleScript.Core.Runtime`

`RuleScriptHostFunctionOptions` provides an extensible alternative to registration overloads with growing positional parameter lists:

```csharp
engine.RegisterFunction(
    "FindPlayer",
    args => FindPlayer(args[0]),
    new RuleScriptHostFunctionOptions
    {
        Parameters = [new RuleScriptParameterSymbol("id", RuleScriptValueType.Number)],
        ReturnType = RuleScriptValueType.String,
        ThreadSafe = true,
        Documentation = "Finds a player by ID."
    });
```

The same options type is accepted by `RegisterFunctionAsync`. `Parameters = null` preserves legacy untyped, variadic behavior; use `Parameters = []` for a typed function with no parameters.

Registered documentation is exposed through `RuleScriptEngine.RegisteredFunctionSymbols` and `RuleScriptAnalysisResult.Functions`:

```csharp
public string? RuleScriptFunctionSymbol.Documentation { get; }
```

`RuleScriptEngine.RegisteredHostFunctions` and `RuleScriptAnalysisResult.HostFunctions` remain available for compatibility with older callers.

Attributed registration can set the same metadata:

```csharp
[RuleScriptFunction(
    Name = "GetStatus",
    Documentation = "Returns the current status.")]
public string GetStatus() => "OK";
```

Attributed methods may return `Task` or `Task<T>` and are automatically registered through the asynchronous Host Function path. `Task<T>` supplies the analyzed return type; `Task` returns `null` to the script. A final `CancellationToken` parameter is supplied by the Engine and is omitted from the script-visible parameter list:

```csharp
[RuleScriptFunction(Documentation = "Waits and returns a value.")]
public async Task<string> EngineDelay(
    double milliseconds,
    CancellationToken cancellationToken)
{
    await Task.Delay(TimeSpan.FromMilliseconds(milliseconds), cancellationToken);
    return "OK";
}
```

Invoke attributed async functions through `ExecuteAsync`; synchronous `Execute` continues to reject async Host Functions.

## Comment and Directive Syntax

| Syntax | Behavior |
|---|---|
| `// text` | Existing single-line comment. |
| `/* text */` | Multi-line comment; may span lines. |
| `/// text` | Documentation metadata for the following function. |
| `#region Name` | Opens a named editor region. |
| `#endregion` | Closes the innermost open region. |

Comments and region directives have no runtime effect.
