# RuleScript v1.10.0 API Reference

This reference covers the public API added or expanded for HostTrigger runtimes in v1.10.0. For editor, formatter, analysis, and host-function APIs introduced earlier, see the v1.8.0 and v1.9.0 release notes and API references.

## `RuleScriptEngine`

Namespace: `RuleScript.Core.Runtime`

### `CreateRuntime`

```csharp
public RuleScriptRuntime CreateRuntime(
    string script,
    RuntimeContext? context = null);
```

Creates a long-running runtime from script text. The returned runtime can be started with `StartAsync`, receive host trigger requests through `TriggerAsync`, and be stopped with `StopAsync`.

### `CreateRuntimeFromFile`

```csharp
public RuleScriptRuntime CreateRuntimeFromFile(
    string path,
    RuntimeContext? context = null);
```

Creates a long-running runtime from a script file. Path resolution follows the same behavior as `ExecuteFile`, including `WorkingDirectory`, `ScriptFileExtension`, `ImportResolver`, and module import loading.

## `RuleScriptRuntime`

Namespace: `RuleScript.Core.Runtime`

```csharp
public sealed class RuleScriptRuntime
{
    public RuntimeContext Context { get; }
    public RuleScriptRuntimeState State { get; }
    public IReadOnlyList<RuleScriptFunctionSymbol> HostTriggers { get; }

    public Task StartAsync(CancellationToken cancellationToken = default);
    public Task StopAsync();
    public ValueTask<RuleScriptTriggerReceipt> TriggerAsync(
        string name,
        IReadOnlyList<object?>? args = null,
        CancellationToken cancellationToken = default);
}
```

`HostTriggers` lists the functions marked with `@HostTrigger`. `StartAsync` begins execution and returns the runtime execution task. `TriggerAsync` enqueues a named trigger request and returns a receipt containing the trigger name and sequence ID. `StopAsync` cancels the runtime, completes the trigger dispatcher, and waits for execution to finish.

`TriggerAsync` throws when the runtime is not running, the script has no active `trigger task` dispatcher, or the requested HostTrigger name is not registered.

## `RuleScriptTriggerReceipt`

Namespace: `RuleScript.Core.Runtime`

```csharp
public sealed record RuleScriptTriggerReceipt(
    long SequenceId,
    string Name);
```

Returned by `RuleScriptRuntime.TriggerAsync` after a trigger request has been accepted into the runtime dispatcher queue.

## `RuleScriptDebugSession`

Namespace: `RuleScript.Core.Runtime`

```csharp
public RuleScriptRuntime CreateRuntime(
    string script,
    RuntimeContext? context = null);

public RuleScriptRuntime CreateRuntimeFromFile(
    string path,
    RuntimeContext? context = null);
```

Creates runtimes configured with the debug session's event handling. Breakpoints, pause events, stepping, continue, and snapshots can be used while the runtime is executing.

## `RuleScriptFunctionSymbol`

Namespace: `RuleScript.Core.Runtime`

```csharp
public RuleScriptHostTriggerMetadata? HostTriggerMetadata { get; }
```

When a user function is marked with `@HostTrigger`, `HostTriggerMetadata` describes the host-visible trigger. Functions without a HostTrigger attribute return `null`.

## `RuleScriptHostTriggerMetadata`

Namespace: `RuleScript.Core.Runtime`

```csharp
public sealed record RuleScriptHostTriggerMetadata(string Name);
```

Stores the trigger name declared in `@HostTrigger("Name")`.

## `RuleScriptAnalysisResult`

Namespace: `RuleScript.Core.Runtime`

```csharp
public IReadOnlyList<RuleScriptFunctionSymbol> HostTriggers { get; }
```

Returns the analyzed function symbols that are marked as HostTriggers. This lets hosts discover trigger entry points without starting a runtime.

## HostTrigger Script Shape

A long-running HostTrigger runtime usually combines one or more `@HostTrigger` functions with a `trigger task` dispatcher:

```rulescript
@HostTrigger("LotArrived")
function OnLotArrived(lotId: string) -> void:
    Print("lot arrived: " + lotId);
    return;
endfunction

parallel:
    trigger task:
        dispatch;
    endtask
endparallel
```

The dispatcher waits for host requests and continues waiting after each dispatch until the runtime is stopped.
