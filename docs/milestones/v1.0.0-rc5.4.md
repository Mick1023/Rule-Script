# RuleScript v1.0.0-rc5.4 Milestone

## Release Goal

Add an explicit debug stop flow so hosts can terminate a `RuleScriptDebugSession` while a script is running or paused at a breakpoint / step pause.

## Completion Summary

- [x] Phase 0 - Stop API Design
- [x] Phase 1 - Pause Cancellation Semantics
- [x] Phase 2 - Running Script Cancellation
- [x] Phase 3 - Runtime Handler Restoration
- [x] Phase 4 - Documentation And Release Readiness

## Phase 0 - Stop API Design

### Goal

Define a public stop API for editor and UI hosts.

### Proposed API

```csharp
session.Stop();
```

Optional future API if needed:

```csharp
await session.StopAsync();
```

### Tasks

- [x] Add `RuleScriptDebugSession.Stop()`.
- [x] Make `Stop()` idempotent.
- [x] Allow `Stop()` before, during, or after a run.
- [x] Define whether stopped runs complete as `OperationCanceledException`.

### Exit Criteria

- [x] UI hosts can bind a Stop button to one method.
- [x] Calling `Stop()` multiple times is safe.

## Phase 1 - Pause Cancellation Semantics

### Goal

Stop a debug run even when execution is currently blocked inside a breakpoint or step pause.

### Tasks

- [x] Release the internal pause wait when `Stop()` is called.
- [x] Clear `IsPaused`.
- [x] Clear `CurrentPause`.
- [x] Make `WaitForPauseAsync` complete or cancel predictably when the session is stopped.
- [x] Ensure a paused `RunAsync` / `RunFileAsync` does not hang after `Stop()`.

### Exit Criteria

- [x] `Stop()` works while paused at a breakpoint.
- [x] `Stop()` works while paused after `StepOver`.
- [x] `WaitForPauseAsync` does not wait forever after stop.

## Phase 2 - Running Script Cancellation

### Goal

Stop a debug run while the script is actively executing.

### Tasks

- [x] Own an internal `CancellationTokenSource` for each debug run.
- [x] Link the host-provided cancellation token with the internal stop token.
- [x] Pass the linked token into `ExecuteAsync` / `ExecuteFileAsync`.
- [x] Propagate cancellation to async host functions.
- [x] Keep existing host-provided cancellation behavior.

### Exit Criteria

- [x] `Stop()` cancels active async waits such as `Delay`.
- [x] `Stop()` stops long-running loops at cancellation check points.
- [x] Existing `RunAsync(..., cancellationToken)` behavior remains compatible.

## Phase 3 - Runtime Handler Restoration

### Goal

Always restore runtime handlers after normal completion, errors, cancellation, or stop.

### Tasks

- [x] Restore `RuntimeEventHandler`.
- [x] Restore `RuntimeEventHandlerAsync`.
- [x] Restore handlers when stopped during a pause.
- [x] Restore handlers when stopped during an async host function.

### Exit Criteria

- [x] Hosts do not lose their runtime event handlers after stop.
- [x] Later non-debug `ExecuteAsync` calls use the original handlers.
