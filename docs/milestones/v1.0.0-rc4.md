# RuleScript v1.0.0-rc4 Milestone

## Release Goal

Add non-blocking script execution support for host-controlled waits and I/O, while preserving the existing synchronous API.

## Completion Summary

- [x] Phase 0 - Async Runtime API Design
- [x] Phase 1 - Async Host Function Registry
- [x] Phase 2 - Async Interpreter Execution
- [x] Phase 3 - Cancellation Support
- [x] Phase 4 - Debug Session Integration
- [x] Phase 5 - Documentation And Release Readiness

## Phase 0 - Async Runtime API Design

### Goal

Define async execution without breaking existing `Execute` and `ExecuteFile` hosts.

### Tasks

- [x] Keep synchronous APIs unchanged.
- [x] Add `ExecuteAsync` overloads for script text.
- [x] Add `ExecuteFileAsync` overloads for files.
- [x] Use `CancellationToken` for async execution.

### Exit Criteria

- [x] Existing synchronous callers continue to compile.
- [x] UI hosts can await script execution directly.

## Phase 1 - Async Host Function Registry

### Goal

Allow hosts to register awaitable functions for delays, I/O, and external waits.

### Tasks

- [x] Add `RegisterFunctionAsync`.
- [x] Pass evaluated arguments to async host functions.
- [x] Pass cancellation tokens to async host functions.
- [x] Keep `UnregisterFunction` and `ClearFunctions` consistent for sync and async functions.

### Exit Criteria

- [x] `Delay` can be implemented with `Task.Delay`.
- [x] Async host function return values can be assigned and used in expressions.

## Phase 2 - Async Interpreter Execution

### Goal

Await async host functions through normal statement and expression evaluation.

### Tasks

- [x] Add async statement execution.
- [x] Add async expression evaluation.
- [x] Add async user function invocation.
- [x] Allow async execution to call existing synchronous host and built-in functions.

### Exit Criteria

- [x] Async host functions work inside assignments.
- [x] Async host functions work inside user functions.
- [x] Synchronous host functions still work during async execution.

## Phase 3 - Cancellation Support

### Goal

Let hosts stop async script execution cleanly.

### Tasks

- [x] Check cancellation before statements.
- [x] Check cancellation inside loops.
- [x] Pass cancellation to async host functions.
- [x] Preserve runtime diagnostics for normal script errors.

### Exit Criteria

- [x] Cancelling an async host wait cancels script execution.
- [x] Long-running loops observe cancellation between iterations.

## Phase 4 - Debug Session Integration

### Goal

Use async runtime execution from debug sessions without removing background isolation for pause/resume.

### Tasks

- [x] Update `RunAsync`.
- [x] Update `RunFileAsync`.
- [x] Preserve breakpoint and step pause behavior.

### Exit Criteria

- [x] Debug sessions can run scripts that use async host functions.
- [x] Breakpoint/step waits do not block the UI thread.
