# RuleScript v1.0.0-rc5 Milestone

## Release Goal

Add awaitable runtime notifications for async hosts, while keeping synchronous runtime notification behavior explicit and unchanged.

## Completion Summary

- [x] Phase 0 - Async Notification API Design
- [x] Phase 1 - Async Interpreter Notifications
- [x] Phase 2 - Compatibility Rules
- [x] Phase 3 - Documentation And Release Readiness
- [x] Phase 4 - Editor Autocomplete Surface
- [x] Phase 5 - Static Script Analysis
- [x] Phase 6 - Best-Effort Script Analysis

## Phase 0 - Async Notification API Design

### Goal

Allow hosts using `ExecuteAsync` or `ExecuteFileAsync` to await runtime notifications.

### Tasks

- [x] Add `RuntimeEventHandlerAsync`.
- [x] Include `CancellationToken` in the async handler signature.
- [x] Preserve existing `RuntimeEventHandler`.

### Exit Criteria

- [x] Async hosts can await UI dispatch, logging, or external tooling work from runtime events.
- [x] Existing synchronous notification hosts continue to compile.

## Phase 1 - Async Interpreter Notifications

### Goal

Route async execution notifications through the awaitable handler.

### Tasks

- [x] Await current-line notifications.
- [x] Await `Print` notifications.
- [x] Await breakpoint and step pause notifications.
- [x] Await runtime error notifications.

### Exit Criteria

- [x] Async runtime event handlers can delay script execution until awaited work completes.
- [x] Async runtime event handlers can control step execution.

## Phase 2 - Compatibility Rules

### Goal

Make sync/async API combinations predictable.

### Tasks

- [x] Keep `Execute` compatible with `RegisterFunction`.
- [x] Make `Execute` reject `RegisterFunctionAsync` functions with a clear runtime error.
- [x] Allow `ExecuteAsync` to call functions registered with `RegisterFunction`.
- [x] Keep `Execute` limited to `RuntimeEventHandler` and ignore async-only runtime event handling.
- [x] Allow `ExecuteAsync` to fall back to `RuntimeEventHandler` when no async handler is configured.

### Exit Criteria

- [x] Hosts can migrate function registration and runtime notifications incrementally.
- [x] Synchronous execution still runs when async-only runtime event handling is configured.

## Phase 4 - Editor Autocomplete Surface

### Goal

Expose stable name snapshots that editor integrations can use for autocomplete.

### Tasks

- [x] Add `RuleScriptEngine.RegisteredFunctionNames`.
- [x] Include both sync and async host function registrations.
- [x] Add `RuntimeContext.VariableNames`.
- [x] Add `RuleScriptEngine.GetVariableNames(RuntimeContext context)`.

### Exit Criteria

- [x] Editors can list registered host functions without executing a script.
- [x] Editors can list variables currently declared in a runtime context.
- [x] Returned lists are snapshots and do not mutate internal state.

## Phase 5 - Static Script Analysis

### Goal

Let editors collect autocomplete symbols from script text before execution.

### Tasks

- [x] Add `RuleScriptEngine.Analyze(string script)`.
- [x] Add `RuleScriptAnalysisResult`.
- [x] Collect script variable names.
- [x] Collect user-defined function names.
- [x] Include registered host function names.
- [x] Include built-in function names.
- [x] Collect import aliases.

### Exit Criteria

- [x] Editors can list symbols from parseable script text without executing host functions.
- [x] Static analysis works before `Execute` or `ExecuteAsync`.
- [x] Runtime context variable APIs remain available for post-execution state.

## Phase 6 - Best-Effort Script Analysis

### Goal

Let editors keep autocomplete available while users are typing incomplete or invalid script text.

### Tasks

- [x] Add `RuleScriptEngine.TryAnalyze(string script)`.
- [x] Add `RuleScriptAnalysisAttempt`.
- [x] Add `RuleScriptDiagnostic`.
- [x] Return diagnostics without throwing for syntax errors.
- [x] Return best-effort symbols when full parsing fails.

### Exit Criteria

- [x] Incomplete scripts return partial symbols and diagnostics.
- [x] Lexer errors return partial text-scanned symbols and diagnostics.
- [x] `Analyze` remains the strict API for parseable scripts.
- [x] `TryAnalyze` does not prevent later `Execute` or `ExecuteAsync`.
- [x] `TryAnalyze` does not prevent later runtime event notifications.
