# RuleScript v1.0.0-rc3 Milestone

## Release Goal

Ship boolean `and` / `or` expression support for `1.0.0-rc3`, including multi-condition `if` statements.

## Completion Summary

- [x] Phase 0 - Boolean Operator Design
- [x] Phase 1 - Lexer Support
- [x] Phase 2 - Parser Precedence
- [x] Phase 3 - Runtime Evaluation
- [x] Phase 4 - If Condition Validation
- [x] Phase 5 - Documentation And Release Readiness

## Phase 0 - Boolean Operator Design

### Goal

Define `and` / `or` behavior consistently with existing comparison and equality expressions.

### Tasks

- [x] Use keywords `and` and `or`.
- [x] Require bool operands.
- [x] Define `and` precedence higher than `or`.
- [x] Keep equality and comparison precedence higher than boolean operators.

### Exit Criteria

- [x] `a == 1 and b == 2 or c == 3` has predictable grouping.
- [x] Non-bool operands fail with `RuntimeException`.

## Phase 1 - Lexer Support

### Goal

Tokenize `and` and `or` as boolean operators.

### Tasks

- [x] Add `TokenType.And`.
- [x] Add `TokenType.Or`.
- [x] Add lexer keywords for `and` and `or`.
- [x] Add lexer tests.

### Exit Criteria

- [x] `and` and `or` are not parsed as identifiers.

## Phase 2 - Parser Precedence

### Goal

Parse boolean operators with stable precedence.

### Tasks

- [x] Add `ParseOr`.
- [x] Add `ParseAnd`.
- [x] Keep equality/comparison as operands of `and`.
- [x] Add parser precedence tests.

### Exit Criteria

- [x] `and` binds tighter than `or`.
- [x] Comparisons bind tighter than `and` / `or`.

## Phase 3 - Runtime Evaluation

### Goal

Evaluate boolean operators at runtime.

### Tasks

- [x] Evaluate `and` with bool operands.
- [x] Evaluate `or` with bool operands.
- [x] Throw `RuntimeException` when either operand is not bool.
- [x] Add runtime tests.

### Exit Criteria

- [x] `true and false` returns `false`.
- [x] `false or true` returns `true`.
- [x] Invalid operands include the operator token in diagnostics.

## Phase 4 - If Condition Validation

### Goal

Allow `if` statements to combine multiple bool expressions.

### Tasks

- [x] Add `if` tests for `and`.
- [x] Add `if` tests for `or`.
- [x] Add `if` tests for mixed `and` / `or`.

### Exit Criteria

- [x] `if enabled and ready then:` works.
- [x] `if enabled or ready then:` works.
- [x] Mixed boolean expressions follow parser precedence.

## Phase 5 - Documentation And Release Readiness

### Goal

Prepare RC3 documentation and validation.

### Tasks

- [x] Update README feature list and examples.
- [x] Add `docs/releases/v1.0.0-rc3.md`.
- [x] Update package version to `1.0.0-rc3`.
- [x] Run the full test suite.

### Exit Criteria

- [x] README documents `and` / `or`.
- [x] RC3 release notes document behavior and validation.
- [x] Full test suite passes.
