# RuleScript v1.0.0-rc5.5 Milestone

## Release Goal

Add a `RuleScriptEngine.Stop()` hotfix so hosts can cancel the currently running engine execution without owning a `RuleScriptDebugSession`.

## Completion Summary

- [x] Phase 0 - Engine Stop API
- [x] Phase 1 - Async Execution Cancellation
- [x] Phase 2 - Sync Execution Cancellation
- [x] Phase 3 - Debug Pause Compatibility
- [x] Phase 4 - Documentation And Release Readiness

## Phase 0 - Engine Stop API

### Goal

Expose a direct engine-level stop API.

### API

```csharp
engine.Stop();
```

### Tasks

- [x] Add `RuleScriptEngine.Stop()`.
- [x] Make `Stop()` idempotent.
- [x] Allow `Stop()` before, during, or after execution.

## Phase 1 - Async Execution Cancellation

### Goal

Cancel active async engine execution and async host functions.

### Tasks

- [x] Own an internal stop token for each `ExecuteAsync` / `ExecuteFileAsync` call.
- [x] Link the internal stop token with host-provided cancellation tokens.
- [x] Pass the linked token into the interpreter.
- [x] Preserve existing host-provided cancellation behavior.

## Phase 2 - Sync Execution Cancellation

### Goal

Allow `Stop()` to interrupt synchronous execution at safe interpreter boundaries.

### Tasks

- [x] Pass the engine stop token into synchronous interpreter execution.
- [x] Check cancellation before statements.
- [x] Check cancellation inside `while` and `foreach`.
- [x] Check cancellation inside user-function bodies.

## Phase 3 - Debug Pause Compatibility

### Goal

Ensure `engine.Stop()` can release a debug run paused at a breakpoint or step pause.

### Tasks

- [x] Make debug pause waits observe the execution cancellation token.
- [x] Clear pause state when cancellation releases the pause wait.
- [x] Keep `RuleScriptDebugSession.Stop()` behavior compatible.

## Phase 4 - Documentation And Release Readiness

### Goal

### Tasks
