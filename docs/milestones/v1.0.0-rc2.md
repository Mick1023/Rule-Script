## Release Goal

Ship the `1.0.0-rc2` runtime with host-observable execution, reliable diagnostics, and interactive debug-session support.

- [x] Main-file and imported-file errors report the correct file and line when available.

## Phase 8 - Interactive Debug Session

### Goal

Provide a host-friendly execution wrapper that can pause execution and wait for host commands.

### Tasks

- [x] Add `RuleScriptDebugSession`.
- [x] Add `RunAsync` for script text.
- [x] Add `RunFileAsync` for file execution.
- [x] Add `WaitForPauseAsync`.
- [x] Add `Continue`.
- [x] Add `StepOver`.
- [x] Expose `IsPaused`, `CurrentPause`, and `Context`.

### Exit Criteria

- [x] Execution can run on a background task.
- [x] Host code can await a breakpoint or step pause.
- [x] Host code can continue or step execution from outside the interpreter callback.

## Phase 9 - Debug Session Runtime Readiness

### Goal

Provide the runtime behavior needed by host applications that want folder-based execution, breakpoints, and step-over control.

### Tasks

- [x] Support selected-folder execution through `RuleScriptEngine.WorkingDirectory`.
- [x] Support `RunFileAsync("main.rules")`.
- [x] Support breakpoint pause and continue in `main.rules`.
- [x] Support breakpoint pause and continue inside imported function files.
- [x] Support step-over after a breakpoint pause.
- [x] Support initial step mode through `RuleScriptEngine.StepExecution`.
- [x] Add debug-session tests for folder-based execution, breakpoints, imported-file breakpoints, and step-over.

### Exit Criteria

- [x] A host can set `WorkingDirectory` to the selected folder.
- [x] The debug session can execute `main.rules` by relative path.
- [x] Breakpoints can be set against `main.rules` and imported files.
- [x] The host can continue execution to completion.
- [x] The host can step to the next executable statement.
