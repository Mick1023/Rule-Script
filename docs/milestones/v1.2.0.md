# v1.2.0

## Goal

Improve realtime code diagnostics and runtime safety while ensuring every analysis and execution uses the latest script and import content.

## Realtime Update Requirements

- [x] Do not introduce compiled-program or cross-execution AST/module caches.
- [x] Analyze the current script text on every `Analyze` and `TryAnalyze` call.
- [x] Resolve current import content on every file analysis and execution.

## Semantic Diagnostics

- [x] Add stable diagnostic codes and `Error`, `Warning`, and `Info` severities.
- [x] Report undefined variables and functions without preventing best-effort analysis.
- [x] Report incompatible known operand and assignment types.
- [x] Report duplicate variable, parameter, and function declarations.
- [x] Allow hosts to provide known external variables so runtime context values are not reported as undefined.
- [x] Preserve source locations and ranges for semantic diagnostics.

## Runtime Limits

- [x] Add an execution timeout with an independent enable switch.
- [x] Add a maximum user-function call depth with an independent enable switch.
- [x] Add a maximum executed-statement count with an independent enable switch.
- [x] Preserve the existing loop-iteration limit and enable switch.
- [x] Apply all enabled limits consistently to sync, async, file, and debug-session execution.
- [x] Include the limit name and configured value in runtime errors.

## Compatibility

- [x] Keep all v1.1.0 public APIs source compatible.
- [x] Keep existing scripts valid unless an enabled runtime limit is exceeded.
- [x] Keep semantic diagnostics advisory; strict execution continues to use runtime behavior.

## Validation

- [x] Add semantic diagnostic code, severity, location, and realtime-update coverage.
- [x] Add enabled and disabled coverage for every runtime limit.
- [x] Add sync, async, file, and debug-session limit coverage.
- [x] Run the full Release build and test suite.
