# v1.0.0-rc5.8

## Goal

Allow hosts to use extensionless imports and choose the file extension used by RuleScript projects.

## Required

- [x] Append `.rules` to extensionless import paths by default.
- [x] Add `RuleScriptEngine.ScriptFileExtension` for host-defined script extensions.
- [x] Apply the configured extension to extensionless `ExecuteFile` paths.
- [x] Apply the same behavior to nested imports and static analysis.
- [x] Preserve paths that already specify an extension.
- [x] Allow `end` to close `if`, `while`, `foreach`, and `function` blocks.
- [x] Preserve the existing block-specific ending keywords.
- [x] Document extensionless project files and custom extensions.

## Validation

- [x] Add coverage for default extensionless imports.
- [x] Add coverage for custom extensions on entry files and nested imports.
- [x] Add coverage for extensionless imports during static analysis.
- [x] Add lexer and nested execution coverage for the generic `end` keyword.
- [x] Run the full test suite.
