# RuleScript v1.0.0-rc5

## Release Candidate Scope

- Adds async runtime event notifications for hosts that need to await UI dispatch, logging, or tooling workflows.
- Documents sync/async compatibility rules for host functions and runtime event handlers.
- Adds editor autocomplete support for registered host functions and runtime variables.
- Adds static script analysis for editor autocomplete before execution.

## Added

- Added `RuleScriptEngine.RuntimeEventHandlerAsync`.
- Added `RuleScriptEngine.RegisteredFunctionNames`.
- Added `RuleScriptEngine.GetVariableNames(RuntimeContext context)`.
- Added `RuleScriptEngine.Analyze(string script)`.
- Added `RuleScriptEngine.TryAnalyze(string script)`.
- Added `RuleScriptAnalysisResult`.

- Added `RuleScriptDiagnostic`.

- Added async runtime notification dispatch for `ExecuteAsync` and `ExecuteFileAsync`.
- Added async notification coverage for current-line changes, `Print`, breakpoints, step pauses, and errors.

## Behavior

- `Execute` can call functions registered with `RegisterFunction`.
- `Execute` reports a clear runtime error if a script calls an async-only function registered with `RegisterFunctionAsync`.
- `ExecuteAsync` can call functions registered with `RegisterFunctionAsync`.
- `ExecuteAsync` can also call synchronous functions registered with `RegisterFunction`.
- `Execute` can use `RuntimeEventHandler`.
- `Execute` ignores `RuntimeEventHandlerAsync`; async event handlers only run during `ExecuteAsync` and `ExecuteFileAsync`.
- `ExecuteAsync` awaits `RuntimeEventHandlerAsync` when it is configured.
- `ExecuteAsync` falls back to `RuntimeEventHandler` when no async handler is configured.
- `RegisteredFunctionNames` returns a sorted snapshot of sync and async host function names.
- `VariableNames` returns a sorted snapshot of variables currently stored in a `RuntimeContext`.
- `Analyze` parses script text without executing it and returns variables, user-defined functions, host functions, built-in functions, and import aliases.
- `Analyze` throws syntax diagnostics for incomplete or invalid script text.
- `TryAnalyze` returns diagnostics and best-effort symbols for incomplete or invalid script text without throwing.

## Example

```csharp
engine.RuntimeEventHandlerAsync = async (runtimeEvent, cancellationToken) =>
{
    await logger.WriteAsync(runtimeEvent.Message, cancellationToken);
    return RuleScriptExecutionDirective.Continue;
};

await engine.ExecuteAsync(script);
```

Editor autocomplete can use the registered function and variable name snapshots:

```csharp
var functions = engine.RegisteredFunctionNames;
var variables = engine.GetVariableNames(context);
```

Static analysis can power autocomplete before execution:

```csharp
var symbols = engine.Analyze(script);

var variables = symbols.VariableNames;
var functions = symbols.FunctionNames;
var imports = symbols.ImportAliases;
```

Best-effort analysis can power autocomplete while the user is still typing:

```csharp
var attempt = engine.TryAnalyze(script);

var variables = attempt.Symbols.VariableNames;
var diagnostics = attempt.Diagnostics;
```
