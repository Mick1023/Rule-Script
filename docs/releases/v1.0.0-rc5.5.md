# RuleScript v1.0.0-rc5.5

## Patch Scope

- Adds an engine-level stop flow with `RuleScriptEngine.Stop()`.

## Added

- Added `RuleScriptEngine.Stop()`.
- Added an internal stop cancellation token for each engine execution.
- Synchronous interpreter execution now observes engine stop at statement, loop, and user-function boundaries.

## Behavior

- `engine.Stop()` cancels the current `ExecuteAsync` / `ExecuteFileAsync` run.
- Async host functions receive cancellation through their `CancellationToken`.
- `engine.Stop()` can cancel `Execute` / `ExecuteFile` when execution is running on another thread and reaches an interpreter cancellation checkpoint.
- `engine.Stop()` releases a `RuleScriptDebugSession` paused at a breakpoint or step pause.
- `RuleScriptDebugSession.Stop()` remains supported and now also requests engine-level stop.
- Stopped runs complete as `OperationCanceledException` or a derived cancellation exception.
- Calling `Stop()` when no run is active is safe.
