# RuleScript v0.7.0

## Added

- Added project file execution with `RuleScriptEngine.ExecuteFile`.
- Added global imports with `import "file.rules";`.
- Added alias imports with `import "file.rules" as alias;`.
- Added module namespace calls such as `robot.GetSensor();`.
- Added alias isolation so alias imports do not pollute the global function table.
- Added circular import detection.

## Syntax

```rulescript
import "common.rules";
import "robot.rules" as robot;

var value = robot.GetSensor();
```

## Import Behavior

- Global imports merge functions into the importing module.
- Alias imports expose functions only through `alias.FunctionName(...)`.
- Nested imports are supported.
- Circular imports throw `RuntimeException`.

## Name Conflict Rules

- Later global imports override earlier global imports with the same function name.
- Functions declared in the current file override globally imported functions.
- Duplicate aliases in the same file throw `RuntimeException`.
- Alias imports are recommended for modules that contain the same function names.
