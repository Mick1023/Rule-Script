# RuleScript v1.0.0-rc5.8

## Added

- Import paths can omit the script file extension; `.rules` is appended by default.
- Added `RuleScriptEngine.ScriptFileExtension` so hosts can use extensions such as `.rule`.
- Extensionless `ExecuteFile` and `ExecuteFileAsync` paths use the configured script extension.
- Added `end` as a shared ending keyword for all block statements.

## Behavior

- Extensionless global, alias, and nested imports all use `ScriptFileExtension`.
- Static analysis resolves extensionless imports with the same rule as execution.
- Paths with an explicit extension are preserved.
- Extension values with or without a leading dot are accepted and normalized.
- `end` can close `if`, `while`, `foreach`, and `function` blocks, including nested blocks.
- `endif`, `endwhile`, `endforeach`, and `endfunction` remain supported.

## Validation

- Added coverage for default and custom script extensions.
- Added coverage for extensionless nested imports and entry files.
- Added static-analysis coverage for custom script extensions.
- Added lexer and nested execution coverage for the generic `end` keyword.
