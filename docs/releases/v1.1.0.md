# RuleScript v1.1.0

## Added

- Added half-open `RuleScriptSourceRange` values for multi-token statement events and diagnostics.
- Added multi-error parser recovery through `Parser.ParseWithDiagnostics()` and `RuleScriptEngine.TryAnalyze()`.
- Added conditional breakpoint overloads to `RuleScriptEngine.AddBreakpoint`.
- Added `RuleScriptDebugSnapshot` with global variables, function locals, and the active call stack.
- Added `RuleScriptDebugSession.CurrentSnapshot` for inspecting a paused run.
- Added inferred variable types through `RuleScriptAnalysisResult.Variables`.
- Added user-function signatures through `RuleScriptAnalysisResult.UserFunctions`.
- Added cursor-aware `Analyze` and `TryAnalyze` overloads with `VisibleVariables`.
- Added optional typed parameters such as `function Add(left: number, right: number):`.
- Added typed `RegisterFunction` and `RegisterFunctionAsync` overloads with parameter and return metadata.
- Added typed Host Function signatures through `RuleScriptAnalysisResult.HostFunctions`.

## Behavior

- Strict parsing and execution continue to throw on the first syntax error.
- Best-effort analysis collects recoverable parser errors and keeps partial symbols.
- Conditional breakpoint expressions run in the current global and local scope and must return a boolean.
- Removing a breakpoint by file and line removes unconditional and conditional breakpoints at that location.
- Untyped function parameters remain backward compatible; typed parameters reject mismatched values before the function body executes.
- Legacy Host Function registrations remain unvalidated; typed registrations validate inputs before invocation and outputs afterward.

## Validation

- Full test suite passed.
- Added coverage for diagnostic recovery, source ranges, conditional breakpoints, local variables, globals, and call stacks.
- Added coverage for type inference, local and imported function signatures, cursor-visible variables, and typed function calls.
- Added coverage for typed synchronous/asynchronous Host Functions, return inference, metadata validation, and legacy registration behavior.
