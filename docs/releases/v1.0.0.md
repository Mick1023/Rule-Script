# RuleScript v1.0.0

## Added

- Added host runtime notifications through `RuleScriptEngine.RuntimeEventHandler`.
- Added awaitable host runtime notifications through `RuleScriptEngine.RuntimeEventHandlerAsync`.
- Added source locations for host-observable runtime events.
- Added current-line notifications.
- Added `Print` host notifications.
- Added breakpoint registration and hit notifications.
- Added step-over execution notifications.
- Added `RuleScriptEngine.Stop()` and `RuleScriptDebugSession.Stop()` for canceling active runs and releasing paused debug sessions.
- Added host error notifications for syntax and runtime failures.
- Added `RuntimeContext.CurrentLocation`.
- Added static analysis and best-effort analysis APIs for editor diagnostics and autocomplete.
- Added autocomplete discovery for host functions, runtime variables, and imported functions.
- Added configurable script file extensions and extensionless script/import paths.
- Added `end` as a shared block-ending keyword while retaining the existing specific ending keywords.

## Behavior

- Synchronous and asynchronous execution support their matching host functions and runtime event handlers.
- Imported functions share the importing program's global runtime context.
- Loop iteration limits can be disabled for long-running workflows while stop and cancellation checks remain active.
- Explicit import extensions are preserved; extensionless paths use `RuleScriptEngine.ScriptFileExtension`.

## Diagnostics

- `RuleScriptException` now exposes `SourceFile` when available.
- Syntax and runtime error notifications include file, line, column, token text, and message when available.

## Validation

- Full test suite passed.
- Added coverage for runtime notifications, async execution, stopping active and paused runs, static analysis, imports, custom script extensions, generic block endings, syntax errors, and imported-file runtime errors.
