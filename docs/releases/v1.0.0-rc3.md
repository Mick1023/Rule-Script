# RuleScript v1.0.0-rc3

## Release Candidate Scope

- Adds boolean `and` / `or` operators.
- Allows `if` and other bool expressions to combine multiple bool conditions with `and` and `or`.

## Added

- Added lexer tokens for `and` and `or`.
- Added parser precedence for boolean expressions.
- Added runtime evaluation for boolean `and` and `or`.
- Added runtime validation that `and` and `or` require bool operands.

## Behavior

- `and` has higher precedence than `or`.
- Comparison and equality expressions bind tighter than `and` and `or`.
- Example: `a == 1 and b == 2 or c == 3` parses as `(a == 1 and b == 2) or c == 3`.

## Validation

- Full test suite passed.
- Added lexer coverage for `and` / `or`.
- Added parser precedence coverage.
- Added runtime coverage for boolean operator results and invalid operands.
- Added `if` condition coverage for `and`, `or`, and mixed `and` / `or`.
