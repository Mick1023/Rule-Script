# RuleScript v1.4.0-rc1.0

## Release Candidate Scope

This release candidate expands RuleScript's data-manipulation syntax and static analysis while preserving the v1.3.0 execution APIs.

```rulescript
var profile = {
    name: "RuleScript",
    tags: ["dsl", "rules"]
};

profile.name = profile?.name ?? "Unknown";
ArrayInsert(profile.tags, 1, "automation");
var [firstTag, secondTag] = profile.tags;

if firstTag == "dsl" then:
    result = profile.name;
elseif secondTag == "automation" then:
    result = "Automated";
else:
    result = "Other";
endif
```

## Changes Since v1.3.0

### Language and Runtime

- Added `elseif` branches with short-circuit evaluation and a single `endif`.
- Added object literals with identifier or quoted property names and nested object/array values.
- Added assignment to object properties, array indexes, nested targets, and writable host-object properties.
- Added conditional member access (`?.`) and right-associative null coalescing (`??`).
- Added array and object destructuring declarations.
- Added `const`, `export function`, and `export const` for explicit module boundaries.
- Modules without any explicit exports retain legacy implicit function visibility for compatibility.

### Built-in Collection Functions

- Added `ArrayInsert(array, index, value)`.
- Added `ArrayRemoveAt(array, index)`.
- Added `ArraySort(array)` for homogeneous number or string arrays.
- Added `ObjectKeys(object)` with ordinally sorted keys.
- Added `ObjectContainsKey(object, key)`.
- Added typed analysis metadata for these built-ins; existing `ArrayRemove` behavior is unchanged.

### Static Analysis and Public API

- Object shapes, array element types, destructured bindings, assignments, nullability, and function return types are now inferred more precisely.
- `RuleScriptFunctionSymbol` now exposes `ReturnType`, `IsReturnTypeNullable`, and `IsExported`.
- `RuleScriptVariableSymbol` now exposes `IsReadOnly` and `IsExported`.
- `RuleScriptAnalysisResult.BuiltinFunctions` exposes typed `RuleScriptBuiltinFunctionSymbol` entries.
- Added diagnostics `RS2008` through `RS2014` for missing properties, invalid/readonly assignments, invalid indexes, null access, invalid null coalescing, and duplicate object properties.
- Added public AST records for object literals, target assignments, null operators, destructuring, and constants.

### Reliability

- Fixed a race in the synchronous stop test without changing public runtime behavior.

## Host API Example

```csharp
using RuleScript.Core.Runtime;

var engine = new RuleScriptEngine();
var analysis = engine.Analyze("""
    function FindName(profile):
        return profile?.name ?? "Unknown";
    endfunction

    var result = FindName({ name: "RuleScript" });
    """);

var findName = analysis.UserFunctions.Single(f => f.Name == "FindName");
Console.WriteLine($"{findName.ReturnType}, nullable: {findName.IsReturnTypeNullable}");

var arrayInsert = analysis.BuiltinFunctions.Single(f => f.Name == "ArrayInsert");
Console.WriteLine($"{arrayInsert.Name} returns {arrayInsert.ReturnType}");
```

## Compatibility

- Existing v1.3.0 host execution APIs remain source-compatible.
- `elseif`, `export`, and `const` are newly reserved words.
- A module that declares at least one explicit export exposes only declarations marked `export`; modules with no exports keep the earlier implicit-function behavior.
- Direct member access on a nullable value remains an error; use `?.` and, when needed, `??`.
