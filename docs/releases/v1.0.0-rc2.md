# RuleScript v1.0.0-rc2

## Release Candidate Scope

- Completed runtime support required by a WPF playground.
- WPF playground UI will be implemented in a separate project.
- This repository now provides the runtime debug/session APIs, tests, and documentation needed by that UI.

## Added

- Added `RuleScriptDebugSession` for host-controlled debug execution.
- Added `RunAsync` and `RunFileAsync` debug-session execution entry points.
- Added `WaitForPauseAsync`, `Continue`, and `StepOver`.
- Added runtime events for current-line changes, `Print`, breakpoint hits, step pauses, and errors.
- Added source location payloads for host-observable events.
- Added breakpoint APIs on `RuleScriptEngine`.
- Added `RuntimeContext.CurrentLocation`.

## WPF Playground Runtime Readiness

- Folder selection can map to `RuleScriptEngine.WorkingDirectory`.
- `main.rules` can be executed through `RunFileAsync("main.rules")`.
- Breakpoints can pause execution in `main.rules`.
- Breakpoints can pause execution inside imported function files.
- Hosts can continue execution after a breakpoint.
- Hosts can step over to the next executable statement.
- `Print` output and syntax/runtime errors are available through runtime events.

## Deferred To Separate WPF Project

- WPF playground UI.
- Folder picker UI.
- Editor/current-line highlighting.
- Breakpoint toggling UI.
- Run, Continue, Step Over, and Stop buttons.
- UI-thread event marshaling and manual UI verification.

## Validation

- Full test suite passed.
- Added debug-session coverage for selected-folder `main.rules` execution.
- Added debug-session coverage for breakpoint pause and continue.
- Added debug-session coverage for imported-file breakpoints.
- Added debug-session coverage for step-over execution.
