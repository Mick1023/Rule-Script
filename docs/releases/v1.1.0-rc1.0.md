# RuleScript v1.1.0-rc1.0

## Release Candidate Scope

This prerelease introduces the v1.1.0 editor diagnostics, debugger integration, and typed function APIs for integration testing before the stable release.

## Added

- Added half-open `RuleScriptSourceRange` values for multi-token statement events and diagnostics.
- Added multi-error parser recovery through `Parser.ParseWithDiagnostics()` and `RuleScriptEngine.TryAnalyze()`.
- Added conditional breakpoints and `RuleScriptDebugSnapshot` pause-state inspection.
- Added inferred variable types, user-function signatures, and cursor-aware visible variables.
- Added optional typed script parameters and typed synchronous/asynchronous Host Function registrations.
- Added Host Function argument, return-value, and metadata validation.

## Compatibility

- Strict parsing and execution continue to throw on the first syntax error.
- Untyped script parameters and legacy Host Function registrations remain supported.
- Existing breakpoint, diagnostic, exception, and runtime-event constructor signatures remain available.

## Validation

- Full Release build and test suite passed.
- Windows and Ubuntu CI validation is required before tagging the release candidate.
- The NuGet package version is `1.1.0-rc1.0`.
