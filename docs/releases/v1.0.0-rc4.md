# RuleScript v1.0.0-rc4

## Release Candidate Scope

- Adds async script execution for hosts that need non-blocking waits, timers, or I/O.
- Adds async host functions while preserving the existing synchronous API.

## Added

- Added `RuleScriptEngine.RegisterFunctionAsync`.
- Added `RuleScriptEngine.ExecuteAsync`.
- Added `RuleScriptEngine.ExecuteFileAsync`.
- Added cancellation token support for async execution and async host functions.
- Added async interpreter execution for statements, expressions, user functions, and module function calls.

## Behavior

- Existing `Execute` and `ExecuteFile` behavior is unchanged.
- `ExecuteAsync` can call async host functions, synchronous host functions, built-in functions, and user-defined functions.
- `Execute` reports a clear runtime error if a script calls an async-only host function.
- `UnregisterFunction` and `ClearFunctions` remove both synchronous and asynchronous host functions.
- Debug sessions now execute through the async runtime while keeping their background task isolation for breakpoint and step pauses.

## Example

```csharp
engine.RegisterFunctionAsync("Delay", async (args, cancellationToken) =>
{
    await Task.Delay(Convert.ToInt32(args[0]), cancellationToken);
    return null;
});

var context = await engine.ExecuteAsync("""
    Delay(1000);
    result = "done";
    """);
```
