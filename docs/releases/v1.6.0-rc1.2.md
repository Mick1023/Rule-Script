# Rule Script v1.6.0-rc1.2

This prerelease adds Attribute-based async Host Function discovery for v1.6.0 testing.

## Async Attribute Scanning

`RuleScriptEngine.RegisterFunctions(host)` now discovers public instance methods marked with `RuleScriptFunctionAttribute` when they return `Task` or `Task<T>`.

```csharp
[RuleScriptFunction(
    Name = "EngineDelay",
    Documentation = "Waits asynchronously.")]
public async Task<string> EngineDelay(
    double milliseconds,
    CancellationToken cancellationToken)
{
    await Task.Delay(TimeSpan.FromMilliseconds(milliseconds), cancellationToken);
    return "OK";
}
```

- `Task<T>` exposes `T` as the Host Function return type.
- `Task` returns `null` to the script.
- A final `CancellationToken` parameter is injected by the Engine and omitted from the script-visible signature.
- Cancellation tokens in other positions produce a registration error.
- Attributed async functions run through `ExecuteAsync`; synchronous `Execute` continues to reject async Host Functions.
- Documentation, parameter metadata, return type, and thread-safety metadata are preserved.

## Compatibility

Existing synchronous Attribute scanning and all manual `RegisterFunctionAsync` overloads remain source compatible.
