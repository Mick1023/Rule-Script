# RuleScript v0.6.1

## Fixed

- Corrected user-defined function assignment scope.
- Function assignments now always write to local function scope.
- Global `RuntimeContext` variables can still be read from functions.

## Breaking Change

In `v0.6.0`, assignment inside a function updated an existing global variable when no local variable existed.

Starting in `v0.6.1`, assignment inside a function never implicitly modifies global `RuntimeContext` variables. To keep script behavior predictable, all function writes are local.

## Scope Rules

Read order:

1. Local function scope
2. Global `RuntimeContext`

Write behavior:

- Function assignment writes local only.
- Function parameters are local.
- `var` declarations inside functions are local.
- Local variables do not leak into `RuntimeContext`.
