# RuleScript v0.1.0

Initial RuleScript release candidate for an embeddable .NET rule engine.

## Features

- .NET 8 class library
- Lexer, parser, AST, and interpreter
- Runtime variable context
- Built-in function library
- Host function registration API
- Structured syntax and runtime diagnostics
- Unit and integration test coverage

## Supported Syntax

- Single-line comments with `//`
- `var` declarations
- Variable assignment
- `number`, `string`, and `bool` values
- Arithmetic operators: `+`, `-`, `*`, `/`, `%`
- Comparison operators: `>`, `>=`, `<`, `<=`, `==`, `!=`
- Grouping with `( expression )`
- Unary operators: `!`, `-`
- `if` / `else` / `endif`
- Function calls

## Built-in Functions

- `Print(value)`
- `ToString(value)`
- `ParseInt(value)`
- `ParseDecimal(value)`
- `Trim(value)`
- `ToUpper(value)`
- `ToLower(value)`
- `Replace(value, oldValue, newValue)`
- `Substring(value, start, length)`
- `Length(value)`

## Host Functions

C# host applications can register custom functions:

```csharp
var engine = new RuleScriptEngine();
engine.RegisterFunction("GetDistance", args => 519);
```

Host functions are resolved before built-in functions and can override built-in names.

## Known Limitations

- No loops
- No arrays
- No JSON/object literals
- No user-defined script functions
- No async host functions
- Parser stops at the first syntax error
- Diagnostics use point locations rather than full source ranges
