# RuleScript v1.0.0-rc5.7

## Added

- Static autocomplete analysis now includes functions from imports.
- Global imports contribute direct function names; alias imports contribute qualified names such as `robot.Read`.
- `RuleScriptEngine.LoopIterationLimitEnabled` can disable the iteration limit for long-running `while` and `foreach` workflows.

## Clarified

- Imported functions share the importing program's global `RuntimeContext` and can create or update values through `global.name`.
- Imported files still cannot declare variables or execute statements at top level.
- Unlimited loops continue to observe engine stop requests and async cancellation between iterations.

## Validation

- Added coverage for imported autocomplete symbols, imported global-variable writes, and sync/async unlimited loops.
