# Rule Script v1.6.0-rc1.1

This prerelease improves Host Function metadata and registration ergonomics for v1.6.0 testing.

## Host Function Documentation

- Adds `RuleScriptHostFunctionSymbol.Documentation` to registered and analyzed Host Function metadata.
- Adds `Documentation` to `RuleScriptFunctionAttribute` for reflection-based registration.
- Preserves Host Function documentation in `RuleScriptAnalysisResult.HostFunctions`.

## Extensible Registration Options

`RuleScriptHostFunctionOptions` groups parameters, return type, thread-safety, and documentation without adding more positional overload parameters. It is supported by synchronous and asynchronous registration.

```csharp
engine.RegisterFunction(
    "FindPlayer",
    args => FindPlayer(args[0]),
    new RuleScriptHostFunctionOptions
    {
        Parameters = [new RuleScriptParameterSymbol("id", RuleScriptValueType.Number)],
        ReturnType = RuleScriptValueType.String,
        ThreadSafe = true,
        Documentation = "Finds a player by ID."
    });
```

Existing registration overloads remain source compatible. `Parameters = null` retains untyped, variadic behavior; `Parameters = []` declares a typed zero-parameter function.
