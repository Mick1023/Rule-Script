# RuleScript v1.9.0-rc1.1

RuleScript v1.9.0-rc1.1 refines strongly typed function diagnostics before the v1.9.0 stable release.

## Fixes

- Old-style functions now suggest an inferred return type when warning about missing return type declarations. For example, a string return suggests `-> string` instead of always suggesting `-> number`.
- Functions with declared return types now warn when one or more parameters are missing type annotations.
- Missing parameter type warnings list the exact untyped parameter names and use singular/plural wording as appropriate.

## Validation

- Strongly typed function diagnostics are covered by focused v1.9 regression tests.
- The full RuleScript test suite is expected to pass for this release candidate.
